# Инструкция по обновлению репозитория GitHub

## Автоматическое обновление

### Windows
Запустите файл `update_repo.bat` двойным кликом или через командную строку:
```cmd
update_repo.bat
```

### Linux/Mac
Запустите скрипт:
```bash
chmod +x update_repo.sh
./update_repo.sh
```

## Ручное обновление

Если автоматические скрипты не работают, выполните команды вручную:

```bash
# Перейдите в директорию проекта
cd "d:\Тимофей\Работа\Go\Calendar"

# Добавьте все изменения
git add .

# Создайте коммит
git commit -m "Обновление: добавлена интеграция frontend-backend, настроен деплой на Vercel, исправлены ошибки TypeScript

- Создан API клиент для связи с backend
- Добавлен хук useItems для управления состоянием  
- Обновлены компоненты для работы с реальным API
- Расширен backend: добавлены методы PUT и DELETE
- Настроена конфигурация TypeScript
- Созданы типы для motion/react
- Добавлена конфигурация Vercel
- Исправлены все ошибки линтера"

# Отправьте на GitHub
git push origin master
```

## Что будет обновлено

- ✅ Все новые файлы (API клиент, хуки, типы)
- ✅ Обновленные компоненты
- ✅ Конфигурация TypeScript
- ✅ Конфигурация Vercel
- ✅ Обновленный backend с новыми методами
- ✅ Документация

## Примечание

Если ветка называется не `master`, а `main`, используйте:
```bash
git push origin main
```
